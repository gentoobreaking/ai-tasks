# T007 - 優化 executor.py GitHub 同步與新增清理功能

## 目標
增強 `executor.py` 對任務狀態的識別能力，防止重複建立 GitHub Issue，並提供清理已關閉任務 Issue 的功能。

## 描述
`executor.py` 的 `--sync-github` 模式原先無法正確讀取非標準格式的 status 欄位，導致已完成的任務被重複建立 GitHub Issue。需要整合 `state_sync.py` 的解析能力，並新增批量關閉 Issue 的機制。

## 完成條件
- [✅] 引入 `read_frontmatter_field` 函數，支援強健的欄位提取
- [✅] 在 `--sync-github` 中改用 `state_sync.read_task_status` 判定狀態
- [✅] 新增 `--cleanup-github` 模式，自動關閉本地已完成但 GitHub 仍開啟的 Issue
- [✅] 修復 `_update_github_issue_url` 在更新時會破壞原始 Markdown 標記的 bug

## 備註
`--cleanup-github` 模式已通過乾跑測試，可識別出 170 個待關閉的 Issue。
