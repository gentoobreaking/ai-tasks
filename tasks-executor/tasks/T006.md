# T006 - 修復專案儀表板進度計算與 Frontmatter 解析

## 目標
修復 `update_projects.py` 與 `update_daily.py` 無法正確解析帶有 Markdown 標記的 Frontmatter 欄位，並優化進度計算邏輯。

## 描述
原本的腳本使用過於嚴格的正則表達式，導致如 `status: done` 格式的狀態無法被識別為 `done`。此外，進度計算未排除已跳過的任務。

## 完成條件
- [✅] 優化 `read_frontmatter` 函數，支援 Markdown 標記（`-`, `**`）
- [✅] 更新 `normalize_status` 支援更多狀態符號（✅, 🔄, ⏭️）
- [✅] 進度計算排除 `skip` 任務，並新增「跳過」欄位顯示
- [✅] 同步修正 `update_daily.py` 的自動日期補全邏輯

## 備註
此修正確保了 PROJECTS.md 與 DAILY.md 的完成率資料準確可靠。
