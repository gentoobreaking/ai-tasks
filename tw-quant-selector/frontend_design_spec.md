# 台股選股系統｜前端設計規格書 v1.0

---

## 0. 設計定位宣言

**受眾**：個人量化研究者，習慣資訊密度高的介面，不需要過度引導。

**設計方向**：`Financial Terminal × Precision Instrument`
— 像交易終端，但比 Bloomberg 更宜人；像科學儀器，每個數字都有上下文。
— 不是面向大眾的投資 App，不需要 onboarding 流程或情感化設計。
— 資訊密度優先，但層次清晰，主動降低認知負荷。

**最讓人記住的一件事**：
> 每個數字旁邊都有它的「脈絡」——排名、歷史分佈、因子貢獻，讓用戶不只看到結果，而是理解原因。

### 2.1 色彩系統

#### 背景層次（深色主題，強制）

```css
:root {
  /* 背景層次（5層）*/
  --bg-base:     #0d0f14;   /* 最底層，頁面背景 */
  --bg-surface:  #141720;   /* 卡片、Panel 背景 */
  --bg-elevated: #1c2030;   /* Hover 狀態、浮層 */
  --bg-overlay:  #242840;   /* Modal、Dropdown */
  --bg-border:   #2a2f42;   /* 分隔線、邊框 */

  /* 文字層次（4層）*/
  --text-primary:   #e8eaf0;  /* 主要文字 */
  --text-secondary: #8b90a4;  /* 次要文字、標籤 */
  --text-muted:     #545870;  /* 禁用、佔位文字 */
  --text-inverse:   #0d0f14;  /* 彩色背景上的文字 */
}
```

#### 語意色彩（核心）

```css
:root {
  /* 上漲（正報酬）*/

  --color-bull-text:   #00e8ac;  /* 文字用（對比度更高）*/

  /* 下跌（負報酬）*/
  --color-bear:        #ff4d6a;  /* 主色 */
  --color-bear-dim:    #3d0014;  /* 低調背景 */
  --color-bear-text:   #ff7088;  /* 文字用 */

  /* 中性 / 警示 */
  --color-neutral:     #f5a623;  /* 橙黃 */
  --color-neutral-dim: #3d2900;

  /* 強調 / 互動 */
  --color-accent:      #4f8ef7;  /* 選取、焦點 */
  --color-accent-dim:  #0a1f4d;

  /* 策略別色彩（因子顏色）*/
  --color-momentum: #a78bfa;  /* 紫 */
  --color-value:    #34d399;  /* 翠綠 */
  --color-quality:  #f59e0b;  /* 琥珀 */
  --color-growth:   #38bdf8;  /* 天藍 */
}
```

#### 漲跌顏色規則

```
所有顯示「報酬率」、「價格變動」、「因子排名變動」的數字：

  值 > 0     → var(--color-bull-text)   + 前綴「+」
  值 < 0     → var(--color-bear-text)   + 前綴無需改（自帶「-」）
  值 = 0     → var(--text-secondary)
  值 = null  → var(--text-muted)  顯示「—」

  背景色標記（整行高亮）：
  大幅上漲（> 5%）→ background: var(--color-bull-dim)
  大幅下跌（< -5%）→ background: var(--color-bear-dim)
```

### 2.2 字體系統

```css
/* 字體組合：數字用等寬，介面用幾何無襯線 */

  --font-ui:      'Syne', sans-serif;           /* 標籤、標題、導航 */
  --font-size-xs: 11px;
  --font-size-sm: 13px;
  --font-size-md: 15px;
  --font-size-lg: 18px;
  --font-size-xl: 24px;
  --font-size-2xl: 32px;
  --font-size-3xl: 48px;
}
```

#### 數字顯示規則

```
所有金融數字必須：
  1. 使用 font-family: var(--font-data)
  2. tabular-nums（等寬數字，讓小數點對齊）
     → font-variant-numeric: tabular-nums;
  3. 固定小數位數（不因數值大小而變動）

格式規範：
  股價       → 整數或兩位小數 (e.g., 543.00)
  報酬率     → 兩位小數 + % (e.g., +12.34%)
  因子分數   → 兩位小數 (e.g., 1.23)
  市值       → 縮寫億元，一位小數 (e.g., 234.5億)
  成交量     → 縮寫萬張 (e.g., 1.2萬張)
  Sharpe等   → 兩位小數 (e.g., 1.85)
```

## 3. 元件規格庫

### 3.1 KPI 卡片（Stat Card）

**用途**：顯示單一關鍵數字，附帶趨勢指示。

**變體**：`default` | `highlight`（主要指標） | `alert`（異常狀態）

```
┌─────────────────────────┐
│ 今日選股數               │  ← label: font-size-sm, text-secondary
│                         │
│  20                     │  ← value: font-size-3xl, font-data, text-primary
│                         │
│ ▲ 較昨日 +2 檔          │  ← delta: font-size-sm, color-bull / color-bear
└─────────────────────────┘

尺寸：最小寬度 180px，高度 120px
```

**必要屬性**：
- `label`: string — 指標名稱
- `value`: string | number — 主要數值
- `delta?`: number — 與前期差異
- `delta_label?`: string — 差異說明文字
- `format`: 'number' | 'percent' | 'currency' | 'raw'
- `variant`: 'default' | 'highlight' | 'alert'

**狀態**：loading（骨架動畫）、error（顯示「—」+ 錯誤色）

### 3.2 選股排行表（Signal Table）

**用途**：顯示當日/歷史選股結果，核心元件。

```
表頭（固定）：
┌──────┬────────────┬────────┬──────────┬──────┬──────┬──────┬──────┬──────────┐
│  排名 │  股票       │  收盤價  │  今日漲跌  │ 動能  │  價值  │  品質  │  成長  │ 綜合分數  │
└──────┴────────────┴────────┴──────────┴──────┴──────┴──────┴──────┴──────────┘

每行：
┌──────┬────────────┬────────┬──────────┬──────┬──────┬──────┬──────┬──────────┐
│  #1  │ 2330 台積電 │ 943.00 │ +2.45%   │ ████ │ ██░░ │ ███░ │ ██░░ │  1.84   │
│  #2  │ 2454 聯發科 │ 1125.0 │ +1.12%   │ ███░ │ ███░ │ ████ │ ███░ │  1.62   │
└──────┴────────────┴────────┴──────────┴──────┴──────┴──────┴──────┴──────────┘

因子欄位使用「迷你橫條圖」：
  - 橫條長度 = 因子 Z-score 映射至 0~100%
  - 橫條顏色 = 各策略識別色（紫/翠/琥珀/藍）
  - 超過 +2σ 的極端值顯示滿格 + 發光效果
  - 懸停顯示精確數值 tooltip
```

**欄位規格**：

| 欄位 | 寬度 | 對齊 | 說明 |
|------|------|------|------|
| 排名 | 48px | 中央 | `#N` 格式，字體較大 |
| 股票 | 160px | 左 | 代號（mono）+ 名稱（ui） |
| 收盤價 | 88px | 右 | font-data，兩位小數 |
| 今日漲跌 | 80px | 右 | 漲跌幅，顏色標記 |
| 動能 | 100px | 中 | 迷你橫條圖 + hover tooltip |
| 價值 | 100px | 中 | 同上 |
| 品質 | 100px | 中 | 同上 |
| 成長 | 100px | 中 | 同上 |
| 綜合分數 | 80px | 右 | font-data，粗體 |
| （選用）持倉狀態 | 60px | 中 | ● 持倉中 ○ 未持倉 |

**行為規格**：
- 排序：點擊任意欄標頭排序，再點切換升/降序
- 點擊行：展開下方 in-line 詳情列（因子趨勢、近30日圖）
- 點擊股票名稱：導航至 `/signals/:stock_id`
- 鍵盤：↑↓ 移動焦點，Enter 展開詳情，Esc 收起
- 每頁顯示 20 筆，不分頁（完整顯示當日所有選出股票）
- ETF 與個股以分組線隔開

## 5. 互動行為規格

### 5.1 載入狀態

```
Skeleton Loading（骨架屏）規則：
  - 所有資料驅動的元件，在資料到達前顯示骨架屏
  - 骨架屏形狀必須與真實內容形狀一致
  - 動畫：左到右流動光暈，週期 1.5s

禁止使用：
  - 旋轉 Spinner（會讓用戶不確定需要等多久）
  - 除非是即時操作回饋（如點擊按鈕後 0.5s 內完成）

資料新舊指示：
  - 若顯示資料超過 24 小時未更新，欄位標頭加黃色警示點
  - Tooltip 說明最後更新時間
```

### 5.2 空狀態

```
選股結果為空時：
  顯示說明文字 + 可能原因清單 + [前往資料監控] 連結
  
  「今日沒有符合條件的選股結果」
  可能原因：
  • 市場休市（今日非交易日）
  • 資料尚未更新（請等待收盤後執行）
  • 選股宇宙篩選條件過嚴
  [查看資料監控]
```

### 5.3 排序與篩選行為

```
表格排序：
  - 預設：依綜合分數降序
  - 排序圖示：▲ 升序 / ▼ 降序 / ── 未排序
  - 最多同時排序 1 個欄位（不支援多欄位排序）
  - 排序狀態保存於 URL query string，可分享 / 書籤

日期選擇器：
  - 只能選擇有資料的交易日（其他日期灰化）
  - 快捷選項：今日 / 昨日 / 上週五 / 上個月最後交易日
  - 鍵盤可操作（← → 切換日期）
```

### 5.4 資料匯出

```
支援匯出格式：
  - CSV（所有表格）
  - JSON（API 格式，含元資料）

匯出觸發：
  - 頁面右上角 [匯出] 按鈕
  - 下拉選擇格式
  - 立即下載，檔名格式：tw_signals_YYYYMMDD.csv

欄位選擇（匯出設定 Modal）：
  - 預設選中：所有顯示中的欄位
  - 可額外加入隱藏欄位（原始因子分數、市值等）
```

### 9.2 API 對接規格

```typescript
// API 回應格式標準
interface ApiResponse<T> {
  data: T;
  meta: {
    generated_at: string;   // ISO 8601
    data_as_of: string;     // 資料截止日期
    request_id: string;
  };
  error?: {
    code: string;
    message: string;
  };
}

// 選股訊號格式
interface Signal {
  stock_id: string;        // "2330"
  stock_name: string;      // "台積電"
  rank: number;
  rank_change: number;     // 與前一期排名差
  close_price: number;
  price_change_pct: number;
  scores: {
    momentum: number;
    value: number;
    quality: number;
    growth: number;
    composite: number;
  };
  is_etf: boolean;
  consecutive_days: number;  // 連續入選天數
}
```

### 9.3 錯誤處理規範

```
API 錯誤統一顯示規則：

  404（無資料）：
    → 顯示空狀態元件（見 5.2）
    → 不顯示 Toast

  500（系統錯誤）：
    → 顯示內嵌錯誤訊息（在元件內部，非 Toast）
    → 提供「重試」按鈕
    → 記錄錯誤至 console

  網路斷線：
    → 頂部顯示「離線中，顯示快取資料」banner
    → 快取資料標示為灰色

  逾時（> 10s）：
    → 取消請求
    → 顯示超時錯誤 + 重試按鈕
```
