## 2. 大師選股策略庫規格

### 2.1 策略清單

共收錄 6 位大師策略，各有不同的哲學核心與台股相容性評估：

| 大師 | 策略風格 | 台股相容性 | 通過標的數（估） |
|------|---------|-----------|----------------|
| 巴菲特 Warren Buffett | 護城河 × 品質 × 合理價格 | ★★★ 高 | ~156 檔 |
| 葛拉漢 Benjamin Graham | 深度價值 × 安全邊際 | ★★☆ 中 | ~87 檔 |
| 彼得林區 Peter Lynch | 成長合理估值（GARP） | ★★★ 高 | ~198 檔 |
| 葛林布拉特 Joel Greenblatt | 神奇公式（EY × ROIC） | ★★★ 高 | 前 30 名 |
| 歐尼爾 William O'Neil | CAN SLIM（動能 × 成長） | ★★★ 高 | ~176 檔 |
| 費雪 Philip Fisher | 成長品質 × 長期持有 | ★★☆ 中 | ~112 檔 |

### 2.2 各策略量化條件規格

---

#### 2.2.1 巴菲特策略

**核心哲學**：以合理價格買進具有持久競爭優勢的優質企業，長期持有。

**量化條件**：

| 條件名稱 | 資料欄位 | 門檻值 | 備注 |
|---------|---------|-------|------|
| ROE（近3年均值） | `financials.roe_ttm` | > 15% | 衡量資本效率 |
| 負債比 | `financials.debt_to_equity` | < 50% | 財務安全邊際 |
| 毛利率 | `financials.gross_margin` | > 30% | 護城河指標 |
| EPS 連續正成長 | `financials.eps_q`（近3年） | 每年 > 0 | 穩定盈餘能力 |
| 本益比 | `valuations.pe_ratio` | < 25 | 允許成長溢價 |
| 自由現金流 | `financials.operating_cf` | 近4季皆 > 0 | 真實獲利能力 |

**建議因子配比**：

```
動能策略  10%   ← 較低，巴菲特不追短期動能
價值策略  30%   ← 確保不過度付價
品質策略  45%   ← 核心，ROE + 護城河
成長策略  15%   ← 確認盈餘趨勢
```

**台股實作注意事項**：
- 「理解得了的生意」是定性條件，量化上以「財務指標穩定性（近4季標準差）」替代
- 半導體代工（台積電、聯電）毛利率門檻可調整為 > 40%
- 護城河最強的台股族群：半導體設備、IC 設計領導廠、電信

---

#### 2.2.2 葛拉漢策略

**核心哲學**：以遠低於內在價值的價格買進，安全邊際保護下行風險。

**量化條件**：

| 條件名稱 | 資料欄位 | 原始門檻 | 台股調整門檻 | 調整原因 |
|---------|---------|---------|------------|---------|
| 本益比 | `valuations.pe_ratio` | < 15 | < 大盤PE × 0.9 | 台股估值結構不同 |
| 本淨比 | `valuations.pb_ratio` | < 1.5 | < 2.0 | 台股PB普遍偏高 |
| 流動比率 | `financials.current_ratio` | > 2 | > 1.5 | 維持財務安全 |
| 長期負債 / 流動資產 | `financials` 計算 | < 1 | < 1 | 不調整 |
| 近10年無虧損 | `financials.eps_q` | EPS > 0 | 近5年 EPS > 0 | 資料深度限制 |
| 穩定配息 | `dividend.dividend` | 近5年皆配息 | 近3年皆配息 | 台股配息文化調整 |

**建議因子配比**：

```
動能策略   5%
價值策略  60%   ← 核心，深度低估
品質策略  25%   ← 財務健全
成長策略  10%
```

**台股實作注意事項**：
- 原始葛拉漢門檻在台股通過率極低（< 5%），建議使用「相對門檻」而非「絕對門檻」
- PB < 1.5 幾乎篩掉全部科技股，使用時需理解選出標的會集中在傳產、電信、金融
- Net-Net 策略（市值 < 淨流動資產）在台股基本不存在，不建議直接套用

---

#### 2.2.3 彼得林區 GARP 策略

**核心哲學**：以合理估值買進高成長公司，PEG < 1 是核心指標。

**量化條件**：

| 條件名稱 | 資料欄位 | 門檻值 | 備注 |
|---------|---------|-------|------|
| PEG 比率 | `valuations.pe_ratio / eps_growth_rate` | < 1 | 成長溢價合理性 |
| 月營收 YoY（近3月均） | `monthly_revenue.revenue_yoy` | > 15% | 台股獨特高頻優勢 |
| EPS 年增率 | `financials.eps_q`（YoY） | > 20% | 獲利成長確認 |
| 本益比上限 | `valuations.pe_ratio` | < EPS成長率 × 100 | GARP 核心規則 |
| 市值規模 | `valuations.market_cap` | 優先 < 1000億 | 小型股成長空間更大 |

**PEG 計算方式**：

```python
# EPS 成長率 = 最近一季 EPS / 去年同季 EPS - 1
eps_growth = (eps_current_q / eps_same_q_last_year) - 1

# PEG = 本益比 / EPS成長率（百分比形式）
peg = pe_ratio / (eps_growth * 100)

# 門檻：PEG < 1 視為合理成長估值
```

**建議因子配比**：

```
動能策略  20%
價值策略  25%   ← PEG 估值約束
品質策略  15%
成長策略  40%   ← 核心，月營收 + EPS 成長
```

**台股獨特優勢**：
- 台灣每月 10 日前公布月營收，是全球少數有此高頻財務資料的市場
- Lynch 最看重的「營收加速成長」可用月營收 YoY 精準捕捉，比美股更有優勢
- 月營收三個月加速成長（YoY 逐月提升）是本策略的強力訊號

#### 2.2.4 葛林布拉特神奇公式

**核心哲學**：同時尋找「便宜」（高盈餘殖利率）且「好」（高資本報酬率）的公司。

**量化條件**：

| 條件名稱 | 計算公式 | 排名門檻 |
|---------|---------|---------|
| 盈餘殖利率（Earnings Yield） | EBIT / 企業價值（EV） | 全市場排名前 20% |
| 資本報酬率（ROIC） | EBIT / (淨營運資金 + 固定資產淨值) | 全市場排名前 20% |
| 神奇公式綜合排名 | EY排名 + ROIC排名 | 合計排名前 30 |

**計算欄位對應**：

```python
# EBIT = 營業利益（operating_income）
ebit = financials.operating_income

# 企業價值（EV）= 市值 + 淨負債
ev = valuations.market_cap + (financials.total_debt - financials.cash)

# 盈餘殖利率
earnings_yield = ebit / ev

# 淨營運資金 = 流動資產 - 流動負債
nwc = financials.current_assets - financials.current_liabilities

# 固定資產淨值 = 不動產廠房設備淨額
net_fixed_assets = financials.ppe_net

# 資本報酬率
roic = ebit / (nwc + net_fixed_assets)
```

**自動排除項目**：
- 金融股（銀行、保險、證券）— 財務結構完全不同，EV 計算無意義
- 公用事業股 — 資本結構特殊，ROIC 失真

**建議因子配比**：

```
動能策略  10%
價值策略  40%   ← 對應盈餘殖利率
品質策略  40%   ← 對應資本報酬率
成長策略  10%
```

---

#### 2.2.5 歐尼爾 CAN SLIM 策略

**核心哲學**：找到盈餘爆發成長、同時有技術面動能確認的股票。

**量化條件（CAN SLIM 七要素）**：

| 字母 | 條件名稱 | 資料欄位 | 門檻值 |
|------|---------|---------|-------|
| C | 當季 EPS 年增率 | `financials.eps_q`（YoY） | > 25% |
| A | 年度 EPS 連續成長 | `financials.eps_q`（近3年） | 每年 > 25% |
| N | 月營收加速成長 | `monthly_revenue.revenue_yoy` | 近3月 YoY 逐月提升 |
| S | 成交量確認 | `daily_prices.volume` | 近4週均量 > 13週均量 × 1.5 |
| L | 相對強弱排名 | 動能因子 z-score | 全市場前 25%（z-score > 0.67） |
| I | 法人持股增加 | （選用，資料取得困難） | 外資近1月淨買超 |
| M | 大盤方向 | 大盤趨勢判斷 | 大盤站上 20MA（系統層級判斷） |

**注意：M 條件（大盤方向）為系統層級設定**，不在個股篩選中處理。建議在排程啟動時判斷，熊市中暫停全部選股。

**建議因子配比**：

```
動能策略  40%   ← 核心，RS 排名 + 成交量
價值策略  10%
品質策略  15%
成長策略  35%   ← EPS + 月營收加速
```

---

#### 2.2.6 費雪策略

**核心哲學**：尋找具長期競爭優勢、優秀管理層的成長型公司，極長期持有。

**量化條件（費雪15要點的可量化部分）**：

| 要點 | 量化對應 | 資料欄位 | 門檻值 |
|------|---------|---------|-------|
| 要點1：銷售成長 | 月營收 5年複合成長率 | `monthly_revenue` | > 15%/年 |
| 要點4：研發費用 | 研發費用 / 營收 | `financials.rd_expense` | > 3%（限科技股）|
| 要點5：利潤率穩定 | 營業利益率近4季標準差 | `financials.operating_margin` | < 3%（穩定性） |
| 要點8：勞資關係 | （無法量化，略） | — | — |
| 要點9：管理層深度 | 董監持股比率 | `TaiwanStockHoldingSharesPer` | > 10% |
| 要點10：成本控制 | 毛利率趨勢（近4季） | `financials.gross_margin` | 逐季提升 |
| 要點15：誠信 | （無法量化，略） | — | — |

**無法量化的要點處理方式**：
- 要點 8（勞資關係）、要點 15（管理層誠信）等定性條件標記為「人工確認項」
- 系統自動評分後，在個股詳情頁提示用戶「此策略有 N 項定性條件需人工判斷」

**建議因子配比**：

```
動能策略  15%
價值策略  10%
品質策略  50%   ← 核心，管理品質 + 利潤率
成長策略  25%   ← 長期銷售成長
```

## 3. 整合模式規格

大師策略可透過三種模式整合至現有四因子系統，互不衝突，可同時啟用。

### 3.1 模式一：篩選器（硬條件過濾）

**運作方式**：
```
原始宇宙（~1,107 檔）
    ↓  大師條件 Boolean 篩選
縮小宇宙（~30～200 檔，依大師而異）
    ↓  四因子 Z-score 評分
最終選股排行
```

**適合策略**：葛拉漢、神奇公式

**規格細節**：
- 每個條件為 `pass/fail` 布林值
- 全部條件通過才進入評分階段
- 篩選器層不計算分數，只做二元判斷
- 宇宙縮小後，四因子評分的相對比較在更純淨的池子中進行
- 每日執行時，篩選層在評分層之前執行

**API 介面**：

```python
class GuruFilter(Protocol):
    """大師策略篩選器介面"""
    
    def apply(self, universe: list[str], as_of_date: date) -> list[str]:
        """
        Input:  選股宇宙（股票代號清單）、計算日期
        Output: 通過篩選的股票代號清單（子集）
        """
        ...
    
    def get_pass_fail(self, stock_id: str, as_of_date: date) -> dict[str, bool]:
        """
        Input:  股票代號、計算日期
        Output: 各條件通過狀態，如 {'roe_15pct': True, 'debt_50pct': False}
        """
        ...
```

### 3.2 模式二：評分因子（第五因子）

**運作方式**：
```
原始宇宙（~1,107 檔）
    ↓  四因子各別 Z-score 評分
    ↓  大師策略評分（0-100 分）
加權合成：
  composite = w1×momentum + w2×value + w3×quality + w4×growth + w5×guru
    ↓
最終選股排行
```

**適合策略**：巴菲特、彼得林區、費雪、CAN SLIM

**評分計算**：

```python

    3. Z-score 正規化至全市場

    Input:  股票代號、大師名稱、計算日期
    Output: 0-100 分（100 = 完美符合，0 = 完全不符）
    """
    ...
```

**權重配置**（五因子版本）：

```yaml

  momentum: 0.25   # 原 0.30，讓出空間給 guru
```

## 4. 四因子架構強化規格

### 4.1 進階參數（各策略可展開調整）

每個策略在頂層權重下，新增可展開的進階參數區塊。

#### 動能策略進階參數

| 參數名稱 | 設定鍵值 | 預設值 | 範圍 | 說明 |
|---------|---------|-------|-----|------|
| 回顧期 | `lookback_long` | 252 天 | 120-504 天 | 252天≈1年（學術標準）|
| 排除近期 | `lookback_short` | 22 天 | 0-63 天 | 排除短期反轉效應 |
| 最少資料天數 | `min_data_days` | 252 天 | 固定 | 低於此天數不計算 |

#### 價值策略進階參數

| 參數名稱 | 設定鍵值 | 預設值 | 範圍 | 說明 |
|---------|---------|-------|-----|------|
| 本淨比上限 | `max_pb` | 30 | 5-50 | 超過視為資料異常 |
| 本益比上限 | `max_pe` | 100 | 20-200 | 虧損股自動排除 |
| 殖利率下限 | `min_yield` | 0% | 0-5% | 0 = 不限制 |

#### 品質策略進階參數

| 參數名稱 | 設定鍵值 | 預設值 | 範圍 | 說明 |
|---------|---------|-------|-----|------|
| ROE 因子權重 | `roe_weight` | 50% | 0-100% | 品質策略內部 ROE 佔比 |
| 槓桿逆指標權重 | `leverage_weight` | 30% | 0-100% | 需與其他子因子合計 100% |
| 毛利率穩定性 | `stability_weight` | 20% | 0-100% | |
| 回顧季數 | `lookback_quarters` | 4 季 | 2-8 季 | |

#### 成長策略進階參數

| 參數名稱 | 設定鍵值 | 預設值 | 範圍 | 說明 |
|---------|---------|-------|-----|------|
| 營收均值月數 | `rev_months` | 3 月 | 1-6 月 | 過濾單月異常訂單 |
| 營收因子權重 | `rev_weight` | 60% | 0-100% | 月營收 vs EPS 佔比 |
| EPS 因子權重 | `eps_weight` | 40% | 自動補足 | |

### 4.2 滑桿鎖定機制

### 6.3 預估交易成本計算

```python
COMMISSION = 0.001425  # 0.1425%
TAX_STOCK  = 0.003     # 0.3%（賣出）
TAX_ETF    = 0.001     # 0.1%（ETF 賣出）
SLIPPAGE   = 0.001     # 0.1% 單向

def estimate_rebalance_cost(
    to_buy: list[str],
    to_sell: list[str],
    portfolio_value: float,
    is_etf_map: dict[str, bool]
) -> CostEstimate:
    per_stock_value = portfolio_value / 20
    
    buy_cost = len(to_buy) * per_stock_value * (COMMISSION + SLIPPAGE)
    sell_cost = sum(
        per_stock_value * (COMMISSION + (TAX_ETF if is_etf_map.get(s) else TAX_STOCK) + SLIPPAGE)
        for s in to_sell
    )
    
    total_cost = buy_cost + sell_cost
    total_pct  = total_cost / portfolio_value
    
    return CostEstimate(
        buy_cost=buy_cost,
        sell_cost=sell_cost,
        total_cost=total_cost,
        total_pct=total_pct
    )
```

## 11. 台股相容性補充說明

### 11.1 需要本地化調整的條件

| 原始條件 | 台股調整 | 原因 |
|---------|---------|------|

- **股東持股揭露**：FinMind 提供週頻率的持股分布資料，可量化大股東持股穩定度
