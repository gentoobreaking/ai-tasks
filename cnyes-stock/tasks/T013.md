# T013 - 歷史檔併發保護

## 目標
防止 `~/.qclaw/cnyes_stock_history.json` 在排程重疊執行時因讀寫競爭導致資料損毀。

## 驗收標準
- [x] `save_history` 採用 `fcntl.flock(LOCK_EX)` + atomic write（暫存檔 → `shutil.move`）
- [x] `load_history` 使用 `fcntl.flock(LOCK_SH)` 共享鎖
- [x] 使用標準函式庫 `fcntl`，無外部依賴
- [x] 鎖只在讀寫期間持有，不影響正常效能

## 備註
- 目前 `save_history` 是 read → modify → write，若兩個 process 同時執行會互相覆蓋
- `fcntl.flock` 在 macOS/Linux 可用，是標準函式庫方案
