# T011 - Trending 區塊偵測邏輯改善

## 目標
改善 `fetch_trending_tw_stock` 中台股區塊的偵測方式，降低對頁面結構的依賴。

## 驗收標準
- [x] 移除「第二個編號區塊 = 台股」的寫法，改為按 rank 歸組 + 關鍵字密度評分
- [x] 策略 1：依 rank 重置 (1→1) 分組，選取關鍵字密度最高的區段
- [x] 策略 2：若無任何區段有匹配，改為個別連結關鍵字比對
- [x] `_guess_section_label()` 能從區段內容猜測區塊名稱
- [x] 兩策略都失敗時輸出清楚警告並回傳空列表
- [x] 已提取輔助函式 `_parse_numbered_links()`、`_tw_score()`、`_detect_tw_section()`、`_guess_section_label()`

## 備註
- 目前依賴「第二個 `1.` 開頭的區塊就是台股」的假設，頁面 layout 一變就斷
- 可考慮先抓取所有區塊標題文字，再匹配「台股」關鍵字來定位
