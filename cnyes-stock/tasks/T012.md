# T012 - Telegram 訊息長度保護

## 目標
處理 Telegram API 4096 字元的訊息長度限制，避免訊息被靜默截斷或發送失敗。

## 驗收標準
- [x] `format_telegram` 移除 `[:20]` 硬截斷，改為動態計算每則訊息 ≤ 4096 字元
- [x] 超過限制時自動分段發送多則訊息（附 `(第 1/3 則)` 標記）
- [x] 若因長度限制而減少發送筆數時，輸出 `[Telegram] 因長度限制，僅發送 N/M 筆新聞`
- [x] `format_telegram_trending` 比照辦理
- [x] `send_telegram` 支援接收字串或字串列表

## 備註
- Telegram API 限制：單則訊息最多 4096 字元（含 HTML tag）
- CLI 輸出不受此限制，僅影響 Telegram 發送
