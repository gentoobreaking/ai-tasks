## 描述

現狀：目前任務是平鋪的清單。
優化：在 Frontmatter 中加入 `blocked_by: T001` 欄位。
* 儀表板優化：在 `PROJECTS.md` 中，如果某個任務的依賴項未完成，則在狀態顯示 `⏳` 或標註「被 T001 阻擋」。這能幫你更清晰地判斷優先順序。

## 需求

1. 定義 Frontmatter 新欄位 `blocked_by`，可以是單一 ID 或 ID 列表（YAML 格式，例如 `blocked_by: [T001, T005]` 或 `blocked_by: T001`）。
2. 修改 `task_status.py`（或讀取 Frontmatter的腳本）以解析此欄位。
3. 更新儀表板生成腳本 (`scripts/update_projects.py`)：在顯示任務狀態時，檢查該任務的 `blocked_by` 列表中是否有任務尚未完成（status 不等於 done）。如果有任何依賴未完成，則在 PROJECTS.md 的任務列表中顯示 `⏳` 或在任務標題後加註「被 Txxx 阻擋」。
4. 確保不會破壞現有任務：如果沒有 `blocked_by` 欄位，則視為無依賴。
5. （可選）提供一個便利指令來添加依賴，例如 `clw block T001 --by T002` 等，但非必須。

## 產出

- 修改後的 Frontmatter 解析腳本（如 `task_status.py`）。
- 修改後的 `scripts/update_projects.py`（或相應的儀表板生成腳本）。
- 在 `ideas2tasks-improvements` 項目的 README 中更新進度。
- 範例：在某個任務的 T*.md 中加入 `blocked_by: T005` 並觀察儀表板顯示變化。

## 驗收標準

- 當任務 A 的 Frontmatter 包含 `blocked_by: T005`，且任務 T005 的狀態不是 done，則在 PROJECTS.md 中任務 A 的狀態顯示為 `⏳` 或有阻擋註記。
- 當任務 T005 完成後，任務 A 的狀態恢復正顯示（不再顯示阻擋）。
- 多個依賴時，只要有任何一個未完成，就顯示阻擋。
- 沒有依賴或依賴皆完成時，不顯示阻擋符號。
- 現有任務沒有 `blocked_by` 欄位也不影響。
