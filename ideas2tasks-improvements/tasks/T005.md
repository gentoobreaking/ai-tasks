## 描述

現狀：需要手動建立 .txt 檔案到 Ideas/。
優化：建立一個極簡的 CLI 指令：`clw idea "今天突然想做個語音助手"`。
* 這個指令會自動生成帶有時間戳的檔案，省去手動建立檔案與切換目錄的麻煩。

## 需求

1. 建立一個新的可執行腳本或命令 `clw idea`（可以放在 `scripts/` 目錄下，例如 `scripts/clw_idea.py` 或 `scripts/idea.py`）。
2. 該指令接受一個字串參數（想法的標題或內容）。
3. 自動生成一個檔案名，格式為 `YYYYMMDD-HHMMSS-<slug>.txt`（或類似現有 Ideas/ 目錄的命名規則），放置在 `~/Tasks/Ideas/` 目錄（或 ideas2tasks 系統使用的 Ideas 目錄）。
4. 檔案內容可以只包含該想法的字串，或包含一個簡單的標題行。
5. 確保不會覆蓋現有檔案：如果同一秒產生衝突，可以加入序號。
6. （可選）提供一個說明訊息，顯示已建立的檔案路徑。
7. 確保指令可以在任何工作目錄下執行（使用絕對路徑）。
8. 可以考慮讓該指令成為一個別名或添加到 shell 的 PATH（但非必須，只要腳本可執行即可）。

## 產出

- 新增的腳本 `scripts/clw_idea.py`（或類似名稱）。
- 在 `ideas2tasks-improvements` 項目的 README 中更新進度。
- 可選：在 `PROJECTS.md` 中顯示此改進的狀態。

## 驗收標準

- 執行 `clw idea "測試想法"` 後，在 `~/Tasks/Ideas/`（或對應目錄）中會出現一個新檔案，檔名包含日期時間戳且內容包含「測試想法」。
- 指令不會產生錯誤（除非參數缺失）。
- 多次執行會產生不同的檔案（不會覆蓋）。
- 檔案內容格式一致且易於後續掃描腳本讀取。
