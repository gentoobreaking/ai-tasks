## 描述

目前的 ideas2tasks 系統主要是「本地推送到 GitHub」。雖然有 `--cleanup-github` 可以關閉 Issue，但如果你在 GitHub 網頁端關閉了一個 Issue，本地的 T*.md 狀態並不會自動變更。

優化：新增 `--pull-github` 模式和 `--reverse-only` 模式。

## 已實作功能

### --pull-github
- 從 GitHub 拉取 CLOSED Issue 並更新本地任務狀態為 done
- 多策略比對：
  - 策略 1：Key 匹配 (project/task_id)
  - 策略 2：正規化標題匹配
  - 策略 3：相似度匹配 (> 0.8)
- 支援描述性任務名稱（如 T001-評估-Fyne-環境）
- 狀態分類輸出：需更新 / 已完成 / Skip / 無法匹配

### --reverse-only
- 從 OPEN GitHub Issues 驅動
- 掃描本地 done/skip 任務
- 自動關閉匹配的 GitHub Issue
- 三層比對策略（同上，相似度閾值 0.85）

## 使用方式

```bash
# 從 GitHub 拉取關閉的 Issue 並更新本地任務
python3 executor.py --pull-github
python3 executor.py --pull-github --dry-run

# 反向同步：從 OPEN Issues 驅動，自動關閉已完成的 Issue
python3 executor.py --reverse-only
python3 executor.py --reverse-only --dry-run
```

## 產出

- 修改後的 `executor.py` 包含：
  - `--pull-github` 參數
  - `--reverse-only` 參數
  - `pull_github_issues_to_local()` 函數
  - `reverse_sync()` 函數

## 驗收標準

- ✅ 當在 GitHub 上關閉一個與本地任務相關的 Issue 時，執行 `--pull-github` 後，本地對應的任務狀態會更新為 done
- ✅ 當本地任務狀態為 done/skip 時，執行 `--reverse-only` 後，會自動關閉對應的 OPEN GitHub Issue
- ✅ 多策略比對可以正確匹配各種格式的任務名稱
- ✅ 輸出會分類顯示：需更新、已完成、Skip、無法匹配的任務數量
